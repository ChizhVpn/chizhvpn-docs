# Contributing

## Ветки
- `staging` — рабочая ветка для тестов
- `main` — прод

Правило: новые изменения → через feature-ветку → PR → staging → промоут в prod.

## Мини-чеклист перед PR
- код запускается локально (`python main.py`)
- миграции актуальны (если менял модели — добавь Alembic migration)
- нет секретов в diff (`.env` не попал, токены не попали)
